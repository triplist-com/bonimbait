# Plan: AI Search Experience Overhaul

> Source PRD: [Issue #10](https://github.com/triplist-com/bonimbait/issues/10)

## Architectural decisions

- **Pre-generated answers**: New `pregenerated_answers` table with vector(1536) for embedding similarity
- **3-tier matching**: keyword overlap (0ms) → embedding cosine (500ms) → Claude stream (2-4s)
- **Category summaries**: Stored as new columns on `categories` table (not a separate table)
- **Shared component**: `ContentSummary` with `variant: 'full' | 'compact'` prop
- **No fake UX**: Silent shimmer → fade-in transition. No step indicators.
- **Budget**: ~$3-4 one-time generation, ~$2-4/day runtime

## Phase 1: ContentSummary Shared Component

**User stories**: #16, #17, #18, #19, #20, #21

### What to build

A polished, reusable React component that renders structured content (summary, key points, costs, tips, warnings) with two visual variants. Extract from VideoPageClient's inline code, improve the design, then replace the inline code with the new component. The video detail page should look better after this change, not just the same.

Design: card-based sections with subtle background tints, section headers with icons, numbered key points with styled circles, green-accented tips, red/amber warnings, improved cost table. Mobile-first responsive layout.

### Acceptance criteria

- [ ] ContentSummary component exists with 'full' and 'compact' variants
- [ ] VideoPageClient uses ContentSummary instead of inline summary/keypoints/costs/tips/warnings
- [ ] Visual improvements visible on video detail pages
- [ ] Compact variant suitable for search answer display
- [ ] Mobile responsive with no horizontal overflow
- [ ] Component tests for both variants with various data combinations

## Phase 2: Pre-Generated Answers Pipeline

**User stories**: #1, #4, #5, #22, #23

### What to build

A one-time pipeline that generates the 200 most likely search questions and their answers. Claude Haiku generates candidate questions from video content, Claude Sonnet generates answers using the compact-context approach (video summaries, not raw segments), and OpenAI generates embeddings for fuzzy matching.

New DB table stores everything. The pipeline is idempotent and can be re-run to add/update answers.

### Acceptance criteria

- [ ] `pregenerated_answers` table created via migration
- [ ] 200 questions generated by Claude based on video content
- [ ] Each question has a structured answer with key_points, costs, tips, warnings, sources
- [ ] Each question has an embedding for similarity matching
- [ ] Total generation cost under $4
- [ ] Pipeline script is idempotent (can re-run safely)

## Phase 3: Answer Matching Service (3-Tier)

**User stories**: #1, #2, #5, #25

### What to build

A backend service that tries 3 matching strategies in order before falling back to live Claude generation. The frontend hook tries the pre-generated endpoint first, renders instantly on hit, falls back to streaming on miss.

New API endpoint for pre-generated lookup. Updated frontend hook to check pre-generated first.

### Acceptance criteria

- [ ] GET /api/answer/pregenerated?q={query} returns match or 404
- [ ] Keyword match works for exact and near-exact queries (<50ms)
- [ ] Embedding match works for paraphrased queries (<500ms)
- [ ] Novel queries fall through to Claude streaming (2-4s)
- [ ] hit_count incremented on each pre-generated match
- [ ] Frontend tries pregenerated first, falls back to streaming seamlessly

## Phase 4: Category AI Summaries

**User stories**: #11, #12, #13, #14, #15

### What to build

Pre-generate an AI summary for each of the 10 categories by analyzing all video summaries in that category. Store in new columns on the categories table. Update the category page to render ContentSummary at the top. Add an admin endpoint to regenerate.

### Acceptance criteria

- [ ] Categories table has ai_summary, ai_key_points, ai_costs_data, ai_tips, ai_warnings columns
- [ ] All 10 categories have AI-generated summaries
- [ ] Category pages render ContentSummary (full variant) at the top
- [ ] Video grid with filtering/sorting remains below the summary
- [ ] Admin endpoint triggers regeneration
- [ ] Generation cost under $1

## Phase 5: Search Loading UX Polish

**User stories**: #6, #7, #8, #9, #10

### What to build

Remove all fake step indicators. Replace with silent shimmer placeholder that preserves layout. AI answer fades in with CSS transition. Video results render immediately. For streaming answers, only show subtle pulsing dots at end of text.

Update AiAnswer component to use ContentSummary (compact variant) for the answer body.

### Acceptance criteria

- [ ] No fake step indicators ("searching", "found", "composing") anywhere
- [ ] Silent shimmer placeholder during AI answer loading
- [ ] CSS fade-in (300ms) when answer appears
- [ ] Video results visible immediately, AI answer loads above them
- [ ] No layout shift when answer appears
- [ ] Pre-generated answers appear instantly (no shimmer visible)
- [ ] Streaming answers show subtle typing dots only
- [ ] Works smoothly on mobile
